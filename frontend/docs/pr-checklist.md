# Pull Request Checklist

Before requesting review, ensure the following are done:

- [ ] Branch is up to date with `main` and rebased if necessary
- [ ] Small, focused changes per PR (one feature/bug at a time)
- [ ] Code follows `docs/coding-guidelines.md`
- [ ] Styles scoped via CSS modules when applicable
- [ ] No console.log left in production code
- [ ] Manual smoke test performed locally (`npm run dev`) and major flows validated
- [ ] Include screenshots or steps to reproduce for UI changes in tickets
- [ ] Add high-level description of the changes in tickets/commits
- [ ] Assign reviewer(s) and add relevant labels in pull requests
