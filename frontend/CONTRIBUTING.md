# Contributing to Frontend

Thanks for helping improve this project! This document points to the team guides that explain how to start, add new features, and follow the repository conventions.

Quick start
- Install dependencies: `npm install`
- Run dev server: `npm run dev` (uses Vite)
- Build for production: `npm run build`
- Preview build: `npm run preview`

Important docs
- `docs/getting-started.md` — how to set up your environment, required tools, and common scripts
- `docs/architecture.md` — high-level project structure and how pages/components/layouts interact
- `docs/coding-guidelines.md` — conventions for components, pages, utilities, styling, and how to use `event.js`
- `docs/pr-checklist.md` — pull request checklist

Branching & PRs
- Create a branch from `main` named `<your-name>-branch` or `feat/<short-description>` or `fix/<short-description>`.
- Keep commits small and focused. Use clear messages.
- Open a PR targeting `main` and add one or more reviewers.
